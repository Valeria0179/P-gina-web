MI BITACORA

Bienvenid@ a mi portafolio virtual de Laboratorio 2. Este espacio ha sido creado con la finalidad de documentar mediante fotos y pequeñas descripciones acerca de las técnicas y conceptos que se aprendieron durante estos tres promedios los cuales conforman todo el ciclo a modo de recordatorio. Espero que esta plataforma te ayude a traer a memoria todo lo visto en clase!.

SESIÓN 1 LABORATORIO

Durante esta primera semana lo que se hizo fue, a manera de introducción, reforzarnos temas vistos en Laboratorio 1, temas como el Arduino y sobre Datos de programación en general. También se nos enseñó un poco sobre cómo inspeccionar una página para leer los códigos de esta. Por último, hicimos una pequeña prueba para estar seguros de haber entendido la clase.

SEMANA 2 - LABORATORIO

Durante esta semana se reforzó el tema de las páginas web, en esta ocasión nos enseñaron sobre cómo editar una plantilla HTML y como editar esta mediante VisualCode Se nos enseñó a cambiar las imágenes dentro de esta plantilla, a cambiar los colores, el texto, etc. Por último, se nos enseñó sobre algunos de los componentes de Arduino.

Volver al inicio

SEMANA 3 - LABORATORIO

Durante esta semana se siguió viendo el tema de la edición de las plantillas en HTML debido a que vamos a trabajar con esto durante todo el ciclo para así recopilar toda la información vista en clase. Se vió también sobre Hosting y sobre el proceso de compra de dominios para así más adelante tener un dominio propio para nuestra página web.

SEMANA 4 - LABORATORIO

Durante esta semana se nos enseñó acerca de Google Sites, sobre cómo personalizar diseños y crear una página de inicio. También sobre cómo interactuar con usuarios mediante las herramientas para agregar contenido.

SEMANA 5 - LABORATORIO

Durante esta última semana de Promedio 1, se nos explicó nuestro proyecto final de promedio, el cual consistía en buscar una solución a una problemática haciendo uso de las ODS, la solución claramente tendría que ser tecnológica. Se nos pidió tambien, junto al proyecto, el crear un prototipo de baja o alta fidelidad o un wireframe para organizar y estructurar nuestra página web la cual tendrá que ser creada más adelante para Promedio 3.

SEMANA 6

Ya entrando al segundo promedio a partir de esta semana, se nos enseñó acerca de los componentes que utilizaríamos para hacer nuestro circuito tecnológico y hacer funcionar a nuestro Arduino UNO, herramientas indispensables como; las resistencias, los cables macho macho, macho hembra, hembra hembra, el protoboard, la placa arduino, entre otros, cabe resaltar que no todos los proyectos utilizan los mismos componentes, algunos varían. También se nos enseñó como referencia, los proyectos de ciclos anteriores.

SEMANA 7

En esta semana se nos enseñó acerca de electrónica básica, y de componentes electrónicos que permiten la circulación de la corriente eléctrica a través de él, como el Diodo, etc. Por último, de manera grupal se acercó nuestro profesor a cardo para aprender a utilizar el multímetro y medir la capacidad de las 'Resistencias', puesto a que las capacidades de cada resistencia varía.

SEMANA 8

En esta semana se nos enseñó a cómo armar nuestro circuito con Arduino UNO de acuerdo al proyecto de cada equipo, en el caso de mi equipo, estamos realizando un teclado para personas con discapacidad en las manos, por lo cual nuestro circuito debía incluir botones personalizados, siendo así, con ayuda del profesor a cargo encontramos un código de botones, lo primero que hicimos fue configurar el botón para que dentro del programa Arduino aparezca que el botón está presionado.

SEMANA 9

Lo primero que hicimos en esta semana fue buscar referencias de teclado para posteriormente plasmar nuestras ideas en un boceto a mano alzada y finalmente pasarlo a 3D. Siguiendo con la semana anterior, luego de crear el código para saber si nuestro botón estaba presionado o no, nuestro profesor a cargo nos mandó a crear una configuración personalizada de nuestro botón, crear un nuevo comando para por ejemplo, hacer que el botón cuando se presione te mande un mensaje, o te lleve a alguna otra pestaña, etc. Tuvimos algunas dificultades para crear el nuevo comando, puesto a que nos dimos cuenta que la placa Arduino UNO no es tan funcional para crear un teclado, pero nuestro profesor nos dió otro tipo de herramientas para poder crearlo de manera efectiva.

SEMANA 10

Durante esta semana se terminó de realizar el prototipo en 3D en el programa 'Blender' de nuestro teclado, para así poder imprimirlo en las impresoras 3D que tenemos dentro de nuestro salón. Para saber si estaba bien ilustrado nuestro prototipo, se separó un teclado para imprimirlo de manera individual, pero al parecer las medidas estuvieron mal pues al momento de imprimir la tecla salió extremadamente grande :c. Por último, se mandó a corregir las medidas para imprimirlo nuevamente la siguiente semana.

PROYECTO FINAL

Este proyecto fue realizado por mi grupo y yo en el curso de laboratorio, donde se desarrolló un teclado especializado para personas con discapacidad motriz. Su objetivo fue crear una herramienta que permita a estas personas acelerar su proceso de trabajo y tener mayor independencia en su vida cotidiana. El teclado cuenta con una interfaz accesible y personalizable para adaptarse a las necesidades individuales de cada usuario. El proyecto fue una experiencia enriquecedora al aplicar los conocimientos adquiridos en el curso para crear una solución innovadora y útil para la sociedad.

PROTOTIPO

Para la creación de este teclado se utilizó el programa 'Blender', después de varios feedbacks por parte del profesor se fue mejorando el prototipo durante cada clase.
